# 文档目录说明

## 目录结构

```
docs/
├── 合规类/          # 证券平台必备，永久保留
│   ├── compliance-guide.md       # 合规指南（数据加密/审计/备份）
│   ├── emergency-handbook.md     # 应急手册（故障处理/数据恢复）
│   └── audit-standard.md         # 审计标准（日志格式/留存要求）
├── 运维类/          # 日常运维用，永久保留
│   ├── deploy-guide.md           # 部署指南（Vercel/Supabase操作）
│   ├── backup-restore.md         # 备份恢复流程
│   └── monitor-guide.md          # 监控告警说明
├── 产品类/          # 业务说明，永久保留
│   ├── operation-manual.md       # 运营手册（管理后台操作）
│   └── user-guide.md             # 用户使用手册
├── 开发类/          # 迭代用，可选保留
│   ├── api-doc.md                # 接口文档
│   └── test-report.md            # 测试报告
└── archive/         # 历史文档归档
```

## 文档分类说明

### 合规类（必须保留）
证券行业监管要求的合规文档，包含数据安全、审计标准、应急预案等。

### 运维类（必须保留）
系统日常运维所需文档，包含部署、备份、监控等操作指南。

### 产品类（必须保留）
面向运营人员和用户的业务文档，包含操作手册和使用指南。

### 开发类（可选保留）
开发过程中的技术文档，可根据需要保留或仅本地存储。

### archive（归档）
历史文档和临时文档的归档目录，不影响主要文档结构。

## 文档维护

- 合规类文档：每季度审查更新
- 运维类文档：每月审查更新
- 产品类文档：功能变更时更新
- 开发类文档：迭代时更新

## 文档责任人

- 合规类：合规部门
- 运维类：运维团队
- 产品类：产品经理
- 开发类：技术团队
